HerdingAgents: A Gamified Sandbox for Learning to Work With AI Coding Agents

June 14, 2026
AI Open Source Education

Boltz — the pixel-art mascot of HerdingAgents, here to coach you through every mission.

The promise of AI coding agents is enormous. Tell an agent "build me a CRUD app" and minutes later you have running code. But the reality of working with these agents day-to-day is more nuanced — they need clear instructions, proper context, well-structured tasks, and a lot of hand-holding. Learning to be effective with AI agents is a skill, and like any skill, it requires practice, feedback, and a safe environment to experiment.

That's exactly what HerdingAgents is designed to provide.

HerdingAgents is a full-stack Next.js application that gives each authenticated user a persistent cloud sandbox (powered by Vercel Sandbox microVMs), a retro-styled desktop interface, and a guided gamified learning path called Agent Ninja. It's an interactive curriculum that teaches you how to move from a novice prompt-jockey to someone who can build real applications, schedule automated workflows, and design what I call "digital employees" — persistent, file-system-aware AI agents that can research, create, build, manage credentials, run on schedules, and integrate with external systems.

The Retro Desktop Experience

When you first log into HerdingAgents, you're greeted by a Windows 95/XP-inspired desktop complete with a taskbar, draggable windows, and a start menu. It's intentionally nostalgic — I wanted the interface to feel playful rather than intimidating. The desktop has icons for the terminal, a workspace file browser, a credential manager, scheduled tasks, and the Agent Ninja learning panel. You can toggle between Windows 95 and Windows XP themes for extra nostalgia points.

Under the hood, every action in this desktop environment maps to real infrastructure. The terminal opens a persistent microVM sandbox. The file browser shows your actual sandbox filesystem. The credential manager encrypts API keys with AES-256-GCM that get injected into your agent's runtime. It's a toy interface with real engineering underneath.

The Agent Ninja Curriculum

The core of HerdingAgents is a 10-level gamified curriculum (plus onboarding) that progressively builds your skills:

  1. Create Files — Learn the basics of instructing an agent to generate project files
  2. Research Topics — Use the agent as a research assistant that documents findings
  3. Install Agent Skills — Extend the agent with community-built skills and tools
  4. Author Custom Skills — Write your own reusable skills for the agent
  5. Build Browser Apps — Guide the agent to build complete web applications
  6. Create Knowledge Bases — Provide reference material so the agent works with your domain
  7. Manage Credentials — Securely handle API keys and secrets in agent workflows
  8. Build Integrations — Connect the agent to external services and APIs
  9. Schedule Automations — Set up recurring agent tasks that run on a timer
  10. Design a Digital Employee — Combine everything to build a persistent, autonomous agent

Each level is guided by Boltz, a pixel-art mascot that acts as challenger, coach, inspector, and narrator. Boltz has multiple poses (standing, sitting, herding, running) and responds to your actions with contextual advice powered by an Anthropic Claude backend. The mascot makes what could feel like a dry tutorial into something genuinely engaging.

Built With OpenCode and Vercel Sandboxes

For the coding agent itself, HerdingAgents uses OpenCode — a CLI-based coding agent framework that runs AI models directly inside the sandbox. OpenCode supports multiple model providers including DeepSeek, Claude, and others, and provides the agent with full filesystem access, command execution, and persistent conversation context. I run it as an npx opencode process inside each user's Vercel Sandbox microVM.

The Vercel Sandbox integration is what makes the whole thing work. Each authenticated user gets their own persistent Linux environment. When you close the browser and come back later, your sandbox is restored from a snapshot — your files, your agent's conversation history, your installed tools. The sandbox supports hibernation to save resources when idle and can be configured to wake up automatically for scheduled tasks.

Beyond Simple Chat

What I find most exciting about HerdingAgents is that it treats coding agents as more than just chatbots. The agent has file system access, can run shell commands, install packages, start dev servers, and manage credentials. The scheduled task system lets you set up recurring prompts that run even when you're not at your computer — your agent can check on deployments, monitor logs, or run daily reports at 8am every morning.

The entire system is instrumented with Langfuse for observability (tracing both Boltz interactions and agent runs) and PostHog for analytics. I wanted to understand how people actually learn to use these agents — where they get stuck, what kinds of prompts work best, which skills are most valuable.

What I Learned Building This

Building HerdingAgents was itself an exercise in working with AI agents. I used OpenCode extensively during development — the very tool that powers the application's backend. There's something beautifully recursive about using an agent to build a platform that teaches people how to use agents.

The hardest part was the sandbox lifecycle management. Making sure each user's microVM is spun up on demand, hibernated when idle, snapshotted regularly, and restored seamlessly requires a surprising amount of orchestration. The Vercel Sandbox API provides the primitives, but the state machine connecting login → sandbox creation → agent execution → backup → teardown took several iterations to get right.

The credential manager was another challenging piece. Storing API keys securely so they're available to the agent at runtime but never exposed in the UI required careful design with AES-256-GCM encryption and strict server-side access controls.

Try It Yourself

The HerdingAgents source code is available on GitHub at github.com/shreysatpathy/herdingAgents. It's a Next.js 16 application with Supabase for auth and database, Tailwind CSS for styling, and a heavy dose of retro UI nostalgia. If you want to try running it yourself, you'll need a Vercel account (for sandboxes), a Supabase project, and API keys for your preferred LLM provider.

Whether you're an experienced developer curious about where agent-assisted coding is headed, or a newcomer trying to understand what all the hype is about, I built HerdingAgents for you. Boltz is waiting.
